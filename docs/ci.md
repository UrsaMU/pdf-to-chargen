# CI / CD Guide

`chargen-extract` is designed to run headlessly in any CI environment.
Pass `--ci` (or set `CHARGEN_CI=true`) to skip all human-in-the-loop checkpoints
and run the full pipeline end-to-end.

---

## Environment variables reference

All settings can be supplied via environment variables. CLI flags take precedence.

### Required (one of these API key groups)

| Variable | Description |
|----------|-------------|
| `ANTHROPIC_API_KEY` | Anthropic API key — uses native Anthropic API |
| `OPENAI_API_KEY` | OpenAI or compatible provider key |
| `OPENAI_BASE_URL` | Custom base URL for OpenAI-compatible providers (optional with `OPENAI_API_KEY`) |

### Pipeline settings

| Variable | CLI flag | Default | Description |
|----------|----------|---------|-------------|
| `CHARGEN_TEXT_DIR` | `--text` | — | Directory of `.txt` files |
| `CHARGEN_OUTPUT_DIR` | `--output` | — | UrsaMU project root |
| `CHARGEN_SYSTEM` | `--system` | — | Game system name |
| `CHARGEN_MODEL` | `--model` | `claude-sonnet-4-6` / `gpt-4o` | Model name |
| `CHARGEN_MAX_CHARS` | `--max-chars` | `150000` | Max characters to read |
| `CHARGEN_CI` | `--ci` | `false` | Auto-approve all checkpoints |
| `CHARGEN_PROVIDER` | `--provider` | auto | `anthropic` or `openai` |

## GitHub Actions

### Basic — Anthropic, triggered manually

```yaml
name: Generate chargen plugin

on:
  workflow_dispatch:
    inputs:
      system:
        description: "Game system name (e.g. 'Shadowrun 5e')"
        required: true
      text_dir:
        description: "Path to .txt files within the repo"
        default: "books/text"

jobs:
  chargen:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: denoland/setup-deno@v2
        with:
          deno-version: v2.x

      - name: Generate chargen plugin
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          deno run --allow-all \
            jsr:@ursamu/chargen-tools/chargen-extract \
            --text "${{ github.event.inputs.text_dir }}" \
            --output . \
            --system "${{ github.event.inputs.system }}" \
            --ci

      - name: Commit generated plugin
        run: |
          git config user.name  "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add src/plugins/chargen/
          git diff --cached --quiet || git commit -m "chore: regenerate chargen plugin for ${{ github.event.inputs.system }}"
          git push
```

### With PDF conversion — full pipeline in one job

```yaml
name: Full chargen pipeline

on:
  workflow_dispatch:
    inputs:
      system:
        description: "Game system name"
        required: true

jobs:
  pipeline:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install poppler + tesseract
        run: sudo apt-get install -y poppler-utils tesseract-ocr

      - uses: denoland/setup-deno@v2
        with:
          deno-version: v2.x

      - name: Convert PDFs to text
        run: |
          deno run --allow-read --allow-write --allow-run \
            jsr:@ursamu/chargen-tools/pdf-to-text \
            books/pdfs books/text --overwrite

      - name: Generate chargen plugin
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          deno run --allow-all \
            jsr:@ursamu/chargen-tools/chargen-extract \
            --text books/text \
            --output . \
            --system "${{ github.event.inputs.system }}" \
            --ci

      - name: Open pull request
        uses: peter-evans/create-pull-request@v6
        with:
          title: "chargen: ${{ github.event.inputs.system }} plugin"
          body: "Auto-generated by chargen pipeline."
          branch: "chargen/${{ github.event.inputs.system }}"
```

### OpenAI-compatible provider (Groq)

```yaml
- name: Generate chargen plugin (Groq)
  env:
    OPENAI_API_KEY:  ${{ secrets.GROQ_API_KEY }}
    OPENAI_BASE_URL: https://api.groq.com/openai/v1
    CHARGEN_MODEL:   llama3-70b-8192
  run: |
    deno run --allow-all \
      jsr:@ursamu/chargen-tools/chargen-extract \
      --text books/text \
      --output . \
      --system "Urban Shadows 2e" \
      --ci
```

## Setting secrets

In your GitHub repository: **Settings → Secrets and variables → Actions → New repository secret**

| Secret name | Value |
|-------------|-------|
| `ANTHROPIC_API_KEY` | Your `sk-ant-...` key |
| `OPENAI_API_KEY` | Your OpenAI or provider key |
| `GROQ_API_KEY` | Groq key if using Groq |

## Local CI simulation

Test your CI setup locally with `act`:

```bash
brew install act
act workflow_dispatch \
  --secret ANTHROPIC_API_KEY=sk-ant-... \
  --input system="Shadowrun 5e"
```

Or run the script directly with env vars inline:

```bash
ANTHROPIC_API_KEY=sk-ant-... \
CHARGEN_CI=true \
deno run --allow-all jsr:@ursamu/chargen-tools/chargen-extract \
  --text ./books/text \
  --output . \
  --system "Shadowrun 5e"
```

## Exit codes

| Code | Meaning |
|------|---------|
| `0` | Pipeline completed successfully |
| `1` | Fatal error (missing required setting, LLM error, file write failed) |
| `1` | Audit failed after 2 retries and `--ci` was NOT set (user aborted) |

With `--ci`, the pipeline never exits `1` due to audit failures — it writes files regardless
(with issues logged to stdout) and exits `0`. Review the generated files before merging.

## Caching the Deno module cache

On GitHub Actions, cache the Deno download cache between runs to speed things up:

```yaml
- uses: actions/cache@v4
  with:
    path: ~/.cache/deno
    key: deno-${{ hashFiles('deno.lock') }}
    restore-keys: deno-
```

## Multi-system matrix

Generate plugins for several game systems in parallel:

```yaml
jobs:
  chargen:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        system:
          - name: "Shadowrun 5e"
            branch: sr5e
          - name: "Urban Shadows 2e"
            branch: us2e
          - name: "Blades in the Dark"
            branch: bitd
    steps:
      - uses: actions/checkout@v4
      - uses: denoland/setup-deno@v2
        with:
          deno-version: v2.x
      - name: Generate ${{ matrix.system.name }}
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          deno run --allow-all \
            jsr:@ursamu/chargen-tools/chargen-extract \
            --text books/${{ matrix.system.branch }}/text \
            --output . \
            --system "${{ matrix.system.name }}" \
            --ci
```
